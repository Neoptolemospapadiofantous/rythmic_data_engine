✅
Oracle VM — C++ only
nq_executor.service runs the C++ binary directly. Zero Python on the VM. No interpreter, no venv, no pip. Binary + JSON config + certs only.
✅
Live UI reads PostgreSQL directly
/api/position · /api/session · /api/trades query Oracle VM's PG in real-time. No EOD sync step. Set PG_HOST=<oracle_ip> in /bot/.env to activate.
ℹ️
SQLite kept for backtests
Historical backtest sessions and ML runs still use bot.db (SQLite). Live session data comes from PG. The two stores serve different purposes.
